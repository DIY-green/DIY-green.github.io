这个 Hack 将介绍一个提升 Activity 启动速度的小技巧。

默认 Activity 的 View 树

Android SDK 中提供了一个用于优化布局的工具 Hierarchy Viewer，我们来试一下用这个工具检测一个默认 Activity 的 View 树。

说明：默认情况下是不能使用 Hierarchy Viewer 来查看真机上的应用的，如果想在真机上使用，可以试试这个：ViewServer。

你会发现新建 Activity 的 ContentView 中指定的 Layout 文件下只有一个 RelativeLayout，该 RelativeLayout 中放了一个 TextView，但是使用 Hierarchy Viewer 一检测发现了很多不可控的节点。

那么这个 Hack 的优化技巧就是通过微调这些节点来提升 Activity 的启动速度。

移除背景

如果你研究过 Activity 的源码，你会发现 FrameLayout 是执行 setContentView() 方法时创建的（这也是当根布局为 FrameLayout 时可以使用 标签来优化的原因）。DecorView 是整个视图树的根节点，是系统创建的。

默认情况下 Activity 是有背景色的（一般是白色），这个 DecorView 就是持有窗口背景图片的视图。如果你的应用需要使用自定义的背景色或者背景图片，那么绘制这个默认的背景色是没有必要的，通过移除背景也可以达到优化的目的。
