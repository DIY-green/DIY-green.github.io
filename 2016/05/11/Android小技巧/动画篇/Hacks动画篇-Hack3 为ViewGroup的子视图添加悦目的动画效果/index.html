这个 Hack 将介绍如何为 ViewGroup 的子视图添加动画效果。

默认情况下，添加到 ViewGroup 中的子视图是直接显示出来的。有一个简单的方法可以为这个过程增加动画效果。

LayoutAnimationController 类简介

LayoutAnimationController 可以为 ViewGroup 的子视图添加动画效果。需要注意的是，并不能为每个子视图分别指定不同的动画效果（也就是说每个子视图都会有相同的动画效果），但是 LayoutAnimationController 可以决定各个子视图显示动画效果的时间。

先看下官方的介绍：

文档上介绍了 ”XML Attributes“，那说明可以在 xml 中使用，也就是说有两种使用方式：直接在 Java 代码中用和在 xml 中定义。在稍微看一下提供了哪些方法：

有几个方法简单解释下：

  • setInterpolator)(Interpolator interpolator)：设置两个子视图之间延迟时间的插值器，注意添加的动画也是可以设置单独设置插值器的
  • setOrder)(int order)：设置控件显示的顺序
  • willOverlap)()：返回两个子视图的动画是否会出现重叠

注意事项

只在第一次设置时有效？
示例中使用了 LinearLayout，通过点击按钮往 LinearLayout 中添加和删除子视图。你会发现在给 LinearLayout 设置好 LayoutAnimationController 后调用添加方法（也就是在 oncreate() 方法中执行的那段代码），会有动画效果。但是，当你再次点击”添加“或”删除“按钮，并没有出现动画效果。

这里没有深入研究了，只是在添加和删除子视图之前又调用了一下 ViewGroup 的 setLayoutAnimation() 方法，但是效果很诡异，所有的子视图都会执行动画。
